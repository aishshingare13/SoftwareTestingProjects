# 🧪 Manual Testing Project

A comprehensive manual testing project repository that contains a structured **Test Plan** and detailed **Test Cases** to ensure the quality, functionality, and reliability of the application under test (AUT).

## 📋 Project Overview

The goal of this project is to define, document, and execute a set of manual tests to validate the core functionalities and non-functional requirements of the application.  
This repository serves as a resource for:

- QA Engineers
- Developers
- Project Managers
- Stakeholders

## 📝 Test Plan

The **Test Plan** defines the following:
- **Scope:** What will and will not be tested  
- **Testing Types:** Functional, regression, usability, integration, etc.  
- **Test Environment:** [Details about the environment, devices, or browsers]  
- **Resources:** Who is responsible for what  
- **Schedule:** Key milestones and timelines  
- **Entry/Exit Criteria:** Conditions for starting and completing testing  
- **Risks and Mitigation:** Known risks and how to address them
  
*Full Test Plan document in the `/TestPlan` folder.*

## ✅ Test Cases

The **Test Cases** provide:
- **Test Case ID**  
- **Title/Description**  
- **Preconditions**  
- **Test Steps**  
- **Expected Results**  
- **Actual Results**  
- **Status (Pass/Fail)**  

*All detailed test cases in the `/TestCases` folder.*


## 🚀 How to Use

1. Review the **Project Overview** for context.  
2. Read the **Test Plan** to understand the approach and scope.  
3. Execute the **Test Cases** as described and record outcomes.  
4. Log any defects in the **BugReports** folder or the project’s issue tracker.  
5. Keep documentation updated with the latest progress.

Technologies & Tools
Documentation: MS Word, Excel (or Google Docs/Sheets)

Bug Tracking: JIRA, Bugzilla (optional)

Test Management: TestRail, Zephyr (optional)
